# GeoPrism Registry

GeoPrism Registry is an open-source platform for curating, governing, and sharing interoperable geospatial knowledge. As a foundational component of a **Geospatial Knowledge Infrastructure (GKI)**, it enables organizations to build high-quality knowledge graphs that model geographic entities, their relationships, and how they change over time, creating a trusted foundation for data integration, decision support, and GeoAI.

Rather than treating geographic data as isolated datasets, GeoPrism Registry manages it as interconnected knowledge. Multiple knowledge graphs can coexist within the platform, each representing different domains, organizations, or perspectives while remaining interoperable through shared semantics and identifiers. The platform also supports publishing curated reference data as flattened datasets, making authoritative information easy to share with systems that are not graph-enabled.

## Why GeoPrism Registry?

Modern geospatial ecosystems depend on data from many organizations, yet geographic entities are often represented inconsistently. Identifiers differ, classifications evolve, relationships are incomplete, and administrative boundaries change over time. These inconsistencies limit interoperability and reduce the quality of downstream analytics and AI.

GeoPrism Registry provides the tools to curate authoritative geospatial knowledge, govern changes over time, and establish interoperable models that can be shared across organizations. It is a foundational component of a Geospatial Knowledge Infrastructure, producing trusted, AI-ready knowledge graphs while enabling organizations to retain sovereignty over both their data and the knowledge that powers their AI systems.

GeoPrism Registry is the first implementation of a Common Geo-Registry (CGR) but its broader purpose is to enable the creation, management, exchange, and publication of interoperable geospatial knowledge for both traditional geospatial applications and next-generation GeoAI.
